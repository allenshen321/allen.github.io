mysql-常用命令总结

发表于 2018-10-03 | 分类于 数据库

mysql常用命令

1. show命令来查看数据库信息
命令 功能
show databases; 显示所有可用的数据库
show tables; 显示当前数据库内可用的所有表
desc tableName; 显示表中的列定义的字段
show status; 显示广泛的服务器状态信息
show create databaseName/tableName; 显示创建数据库/表的Mysql语句
show grants; 显示授予用户的安全权限
show errors/warnings; 显示服务器错误/警告信息
2. 简单查询数据
命令 功能
select columnName1, columnName2… from tableName; 从表中检索列1, 列2…
select * from tableName; 从表中查询所有的列
select distinct 列1 from tableName; 返回列1中,各行的值都不同, 不能部分使用distinct
select 列1, 列2… from tableName limit num1,num2; 返回从num1 开始的num2行, 如果只有一个数,则从第0行开始

注意点

  1. 在select from tableName;语句中,除非你确实需要表中的每个列,否则最好别使用 通配符。虽然使用通配符可能会使你自己省事,不用明确列出所需列,但检索不需要的列通常会降低检索和应用程序的性能。

  2. select 语句使用的distinct,不能部分使用distinct,distinct应用于所有的列而不仅是前置它的列,如果给出select distinct 列1, 列2 from 表名;除非制定的两个列都不同,否则所有的行都被检索出来.

  3. sql语句中可以完全限定一个列和表,只需在列或表名前面指定相应的表或数据库即可,例如:tableName.column1, databaseName.tableName, 限定为tableName中的column1.

3. 排序order by语句

order by 语句对检索出来的数据进行排序,这个子句必须放在selext 语句的最后一条.

语句: 

select 列1, 列2... from tableName order by 列1, 列3 DESC;

解读:

  1. 该句的功能是从表名中选取列1和列2, 在检索出的结果中先按列1进行升序排序,然后在之前排序的基础上再按照列3降序排序.最终的到检索结果.

  2. 如果相对多个列进行降序排序则必须每个列后面都加DESC关键子.表示降序.

4.过滤数据where语句.

where 语句根据写的条件对数据进行过滤,这个子句放在表名(from子句)之后.

句法:

select 列1, 列2... from tableName where 条件 order by lie1, 列3 DESC;

解读:

  1. where语句和order by语句共用时, order by语句放在后面;

  2. where语句中有两个表示不等于的字符’<>’和’!=’,相同点是都表示不等于,不同的是’<>’是sql标准语句,而’!=’是非标准.参考:https://stackoverflow.com/questions/14159569/what-is-the-difference-between-and-operators-in-mysql

  3. 在数据库中有个特殊的NULL,它于字段包含0,空字符串或仅仅包含空格不同.通过它过滤是需要用到特殊的where子句:

    select * from tableName where column is null;

where高级用法

  1. and和or逻辑运算符,

    • 在SQL(像多数语言一样)在处理 OR 操作符前,优先处理 AND 操作符。

    • 任何时候使用具有 AND 和 OR 操作符的 WHERE 子句,都应该使用圆括号明确地分组操作符。不要过分依赖默认计算次序,即使它确实是你想要的东西也是如此。使用圆括号没有什么坏处,它能消除歧义。

  2. in操作符

    • in操作符用来指定条件范围,范围中的每个条件都可以进行匹配.IN 取合法值的由逗号分隔的清单,全都括在圆括号中。

    • 相比or操作符的有点:

      (1)在使用长的合法选项清单时, IN 操作符的语法更清楚且更直观。

      (2)在使用 IN 时,计算的次序更容易管理(因为使用的操作符更少)。

      (3)IN 操作符一般比 OR 操作符清单执行更快。

      (4)IN 的最大优点是可以包含其他 SELECT 语句,使得能够更动态地建立WHERE子句.

  3. not操作符

    not操作符表示否定它之后所跟的条件.

通配符
名称 功能
% 表示任何字符出现任意次数,可以0次, 可以在任意位置使用,也可以使用多个通配符.但是不可以匹配NULL
下划线 _ 匹配单个字符

使用技巧

  1. 不要过度使用通配符。如果其他操作符能达到相同的目的,应该使用其他操作符。

  2. 在确实需要使用通配符时,除非绝对有必要,否则不要把它们用在搜索模式的开始处。把通配符置于搜索模式的开始处,搜索起来是最慢的。

  3. 仔细注意通配符的位置。如果放错地方,可能不会返回想要的数

正则表达式

在mysql中的where子句中,还支持使用正则表达式匹配.这种用法和like相似,只是将关键字换成REGEXP,示例如下:

select * from tableName where 列1 regexp '正则表达式';
匹配结果是列1符合正则的行.
拼接字段concatennate

在mysql的select语句中,可以使用Concat()函数来拼接两个列,显示成帅选结果的一个列中.

示例:

SELECT Concat(列1, ' (', 列2, ')') as 新列名 FROM tableName;

解析:

- 该语句在查询结果中,显示的是一列,该列名为新列名, 数据元素的格式是:列1 (列2).
- Concat()需要1个或多个指定的串,各串之间用逗号分隔.
- 上面的select语句中concat()函数连接了4各元素:列1, ' (', 列2, ')'.
- 在列之后可以使用AS给列另起一个别名,在查询结果中,该列将以新列名显示.
