‘Redis报错：MICONF,原因分析’

发表于 2018-05-24 | 分类于 数据库
出错详情

在用scapy-redis爬虫时，利用reids数据库对数据进行去重，我是利用RedisPipeline去保存数据并去重，但是但数量到达接近500万item时，爬虫程序报错：

redis.exceptions.ResponseError: MISCONF Redis is configured to save RDB snapshots, but is currently not able to persist on disk. Commands that may modify the data set are disabled. Please check Redis logs for details about the error.

错误的大致意思是： redis已经配置为保存rdb快照模式，但是现在不能持久化到硬盘，修改数据集的命令不能使用，错误的详情请查看redis log日志。
原因分析

查看了log日志，信息如下：

redis can not continue. Exiting.
[9372] 24 May 16:46:35.338 # fork operation failed
[9372] 24 May 16:46:35.416 # Background saving terminated by signal 1
[9372] 24 May 16:46:41.104 * 1 changes in 900 seconds. Saving...
[9372] 24 May 16:46:41.182 * Background saving started by pid 12520
[12520] 24 May 16:46:41.307 #
The Windows version of Redis reserves heap memory from the system paging file
for sharing with the forked process used for persistence operations.At this time there is insufficient contiguous free space available in the
system paging file. You may increase the size of the system paging file.
Sometimes a reboot will defragment the system paging file sufficiently for
this operation to complete successfully.

日志说的意思是：windows版本的redis从系统分页文件中保存堆内存，用来共享给持久化操作的分支进程。与此同时，系统分页文件中可用空间不足。你可以增加系统分页文件的大小。有时候重启对系统分页文件碎片整理将是有效的对于操作的成功完成。

日志中提到两个方法：

  1. 增加系统分页大小；
  2. 重启

redis已占内存3.13G 我的电脑内存是8G，应该是内存不足，针对这种情况，要么增加内存，要么清理redis中的数据，因为我直接把item放入redis中了，

解决方案：

这里有两种方法解决这个问题：

  1. 定制scrapy的Pipeline中间件，将数据保存到mysql或mongodb中
  2. 及时将redis数据库中的items转移到mysql或mongodb数据库中

我这里采用第二种方法，写个python脚本，当数据库中有items的时候，将items保存到mongodb中，代码如下：

import json
import redis
import pymongo


def main():
    # 指定Redis数据库信息
    rediscli = redis.StrictRedis(host='127.0.0.1', port=6379, db=0)
    # 指定MongoDB数据库信息
    mongocli = pymongo.MongoClient(host='localhost', port=27017)

    # 创建数据库名
    db = mongocli['mongo_db_name']
    # 创建表名
    sheet_goods = db['mongo_collection_name']
    i = 1
    while True:
        # FIFO模式为 blpop，LIFO模式为 brpop，获取键值
        source, data = rediscli.blpop(["redis_data_name"])
        data = data.decode('utf-8')
        item = json.loads(data)
        sheet_comment.insert(item)


if __name__ == '__main__':
    main()

当运行脚本准本转移数据是，发现还是报错，百度搜索后发现，redis报错后会强制关闭redis持久化的快照功能， 通过查看配置文件，参数‘stop-writes-on-bgsave-error yes’，这里变成了yes，需要我们改成no

修改参数有两种方式：

  1. 直接修改配置文件，但是修改完需要重启redis服务，这样才会生效；

  2. 在redis-cli中，通过命令修改，命令如下：

    127.0.0.1:6379> config set stop-writes-on-bgsave-error no

这样修改后会直接生效。不用重启redis服务。

至此，redis因内存不足报错的问题就解决了

注意

redis报此类问题的原因也有可能是因为设置的maxmemory参数有关，当你发现redis占用内存远远小于系统内存的时候，可以适当增大这个参数，也可以增加redis占用内存。
