flask-工厂函数和蓝本

发表于 2018-09-23 | 分类于 flask

工厂函数

在flask web开发中,通常我们需要创建不同配置环境下的实例,以便能够在不同配置文件下运行程序.这时候工厂函数可以实现这个功能,工厂函数可以返回不同配置的实例,具体返回那个配置文件的实例,由用户传入的不同配置文件参数来决定.

代码示例

  1. 创建不同配置环境下的Flask()实例

    from flask import Flask

    def create_app(‘config_name’)

    app = Flask(__name__)
app.config.from_object('config_name')

return app
    1. 返回的是config_name配置下的Flask()实例app.
    2. 配置文件通过Flask app.config.from_object()方法,直接导入到程序中.

  1. 在工厂函数中加入flask扩展

    from flask import Flask
from flask.ext.Bootstrap import Boostrap

bootstrap = Bootstrap()

def create_app('config_name')
    app = Flask(__name__)
    app.config.from_object('config_name')

    bootstrap.init_app(app)

    # 返回配置好的Flask()实例app
    return app
    1. flask的扩展程序,需要先建立全局的实例,然后在工厂函数中,Flask()程序创建并配置好后,再初始化扩展程序,初始化扩展程序直接调用.init_app()方法即可完成扩展程序的初始化.
    2. **由于Flask实例对象是在函数工厂中创建的,所以这时候创建的实例app是在运行时创建的,只有在调用create_app函数后,才能使用app.route()配置路由,但是此时配置路由就晚了. 为了解决这个问题,Flask的蓝图功能Blueprint需要注册到工厂函数中.

  2. 在工厂函数中注册蓝图.例如:蓝本是main

    from flask import Flask
from flask.ext.bootstrap import Bootstrap

bootstrap = Bootstrap()

def create_app('config_name'):
    app = Flask(__name__)
    app.config.from_object('config_name')

    # 加入扩展
    bootstrap.init_app(app)

    # 注册蓝图
    from .main import main as main_blueprint
    app.register_blueprint(main_blueprint)

    return app
    1. 这样工厂函数的功能就完整了,程序的视图函数在蓝本中定义,其中可以包扩路由和错误文件的定义等等.

flask中的蓝图

蓝图中定义的路由处于休眠状态,直到蓝图注册到程序上后,路由才能真正的成为程序的一部分.一个程序中可以有多个蓝图,用来实现不同功能的路由.

蓝图的创建

蓝图保存在程序包中的一个子包.在子包中的构造文件中创建蓝图.

from flask import Blueprint

main = Blueprint('main', __name__)

from . import views, errors
  1. 通过实例化一个Blueprint()对象来创建蓝图,Blueprint()中有两个参数,一个是蓝图名,一般情况下和蓝图包名一样,第二个参数是蓝图所在的包和模块,一般写python中name变量即可.
  2. 在构造文件的末尾还必须导入路由视图和错误处理程序,以将蓝图和路由,错误处理文件关联起来.
  3. 在末尾导入路由和错误处理文件是为了防止循环导入模块,因为在路由和错误处理程序中也要导入蓝图.

编写错误程序时的不同

在蓝图中编写错误处理程序中, 修饰器要使用app_errorhandler, 而不是errorhandler.两者的区别是:errorhandler的作用域是蓝图中,而app_errorhandler的作用域是整个程序.

from . import main  # main是蓝图名

@main.app_errorhandler(404)
def page_not_found(e):
    ...

编写视图函数时的不同

  1. 修饰器由蓝图提供.
  2. url_for() 函数的第一个参数是端点名,也就是视图函数名,在蓝图中前面要加上蓝图的名字,表示此蓝图中的视图.
