发表于 2018-09-29 | 分类于 网络编程

TCP/IP 协议

协议是指计算机与网络设备要相互通信,双方必须基于相同的方法.比如,如何探测到通信目标,哪一方先发起通信,使用那种语言进行通讯,怎样借宿通信等等规则.不同的硬件.操作系统之间的通信都需要一种规则,这种规则就称为协议(protocol)

TCP/IP协议是互联网相关的各种协议的总称.

分层管理

TCP/IP协议族中最重要的一点就是分层管理.TCP/IP协议按层次分为:网络链路层, 网络层, 传输层, 应用层.

各层次的作用:

  1. 链路层(也称链路层, 网络接口层)

用来处理链接网络的硬件部分.包括控制操作系统,硬件的设备驱动, 网卡, 光纤等物理课件的部分.硬件上的范畴均在链路层的作用范围.

  1. 网络层

网络层用来处理网络上流动的数据包.数据包是网络传输的最小的数据单位.网络层规定了通过怎样的路径(传输路线)达到对方的计算机,并把数据包传送给对方.

  1. 传输层

对上层应用层,提供了处于网络链接中的两台计算机之间的数据传输.

传输层有两个性质不同的协议:TCP(Transmission Control Protocol, 传输控制协议) 和 UDP(User Data Protocol, 用户数据报协议o)

  1. 应用层

应用层决定了向用户提供应用服务时通信的活动.

TCP/IP协议族内预存了各类的通用的应用服务,其中包括了FTP(File Transfer Protocol, 文件传输协议) 和DNS(Domain Name System, 域名系统),等等.

客户端发送请求过程

利用TCP/IP协议族进行网络通信时,通过分层顺序与对方沟通.发送端从应用层向下层走,接受端则相反.

以上图为例,请求分四步:

  1. 发送端在应用层(HTTP协议)发出url的HTTP请求.

  2. 传输层(TCP协议)把应用层接受到的数据(HTTP请求报文)进行分割,并在各个报头打上标记序号及端口号后发送给网络层.

  3. 网络层(IP协议),增加通信目的地MAC地址后转发个链路层.达到这一层,发送网络的通信请求所需要的信息已经准备齐全了.

  4. 链路层负责将请求信息发给接受端的服务器.

发送端在层与层之间传输数据时,没经过一层都会被打上一个该层所属的首部信息.接受端则相反,经过每层会消去所对应的标签.
