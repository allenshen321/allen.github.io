functools.wraps()作用

functools.wraps()函数可以将原函数对象的指定属性复制给包装函数对象,包含module,name,doc或者通过参数选择.常用于装饰器中.

装饰器作用过程

1.两层嵌套

示例:

def log(func):
def wrapper(*args, **kw):
    print('call %s():' % func.__name__)
    return func(*args, **kw)
return wrapper

@log
def now():
    print('2015-3-25')

>>> now()
call now():
2015-3-25

flask web开发-配置选项

发表于 2018-09-11 | 分类于 flask

配置文件

程序经常需要多个配置,比如在开发.测试.生产环境使用不同的数据库,这样才不会影响.

在程序的结构文件中,config.py文件的内容示例如下:

import os

class Config:
    """基类配置文件"""
    pass

class DevelopmentConfig(Config):
    """开发环境配置文件"""
    pass
...

config = {
    'development': DevelopmentConfig,
    ...

    'default': DevelopmentConfig
    }
  1. 配置文件中，可以定义一个基类配置文件，用以定义各环境通用的配置，接下来可以定义不同环境下的配置文件．其继承基类的配置文件．
  2. 配置文件中,如果有些选项比较私密,比如密码类的,可以在环境变量中定义,在配置文件中,从环境变量中获取.
  3. 在配置文件的末尾可以配置不同的配置环境,config字典, 其中注册了一个默认配置.
  4. 配置文件中,配置选项只有全部字母是大写的变量才能被配置对象使用.所以在配置文件中,配置变量务必是全部大写.

  5. 如上例所示,采用基类和类的继承来配置配置选项,那么在导入到app程序时,必须使用from_object()的方式导入配置文件.

    app = Flask(__name__)
app.config.from_object(config['development'])
# 导入开发环境的配置

1.简述函数式编程

在函数式编程中,函数是基本单位,变量只是一个名称而非存储单位.除了匿名函数外,python还使用filter(),map(),reduce(),apply()来支持函数式编程.

  1. filter()函数

用于过滤序列,过滤掉不符合条件的序列,返回由符合条件的序列组成的列表.在python3中,返回的是符合条件的元素的迭代器对象.

filter()函数接受两个参数,第一个是函数名,第二个是序列,序列的每个元素作为函数的参数进行判断,返回True或False,最后将所有结果True的元素返回.

1、生成器

生成器的创建方法：

(1)通过列表生成式创建

可以通过将列表生成式的[]改成()

eg: 
# 列表生成式
L = [ x*2 for x in range(5)]  # L = [0, 2, 4, 6, 8]
# 生成器
G = （ x*2 for x in range(5))  # 此时的G是,<generator object <genexpr> at 0x7f626c132db0>

创建列表生成式和生成器的区别只是最外层的()和[],列表生成式是一个列表，而生成器事宜个可迭代对象。生成器对象可以通过for语句遍历每个元素。

for each in G:
print(each)  # 打印结果是 0 2 4 6 8
