58字体反爬虫的分析

发表于 2019-04-14 | 分类于 爬虫

问题由来

在爬取某网站求职者简历的时候,碰到了提取的信息是乱码,分析源代码发现源代码中要提取的信息是通过 ໟs; 的形式,明显不是我们要的信息.这种网站的反爬虫策略一般是采用了字体反爬虫策略.通过css中添加 @font-face标签,将网站中的某些字用代码表示,在渲染的时候在转换回相应的文字.

解决方案

网站自定义字体,通常用来替换到该网站的核心信息,通常替换字的数量不会太多,可以通过手工提取字体库中所要替换的字体,每个字体会对应一个唯一的unicode码,通过分析要替换的代码替换掉网站源代码中的相关字符就可以了.

工具

python3.6
fontTools 处理网站中woff/ttf字体库文件,可以提取字体库中cmap映射关系,也可以将.woff/.ttf文件转化为.xml文件,看到其映射关系
fontforge 一个可以创建自己的文字库的软件, 此处用来查看.woff/.ttf文字库中的文字

实际操作思路

  1. 分析网站, 找到字体转换的元素, 并找到其中css中用到的@font-face文件.

找到左侧加密的源代码,然后后侧可以看到该元素的属性,其中类名称为stonefont的字体的 font-family 采用的是customefont, 全局所搜这个关键字.

从@font-face标签可以看到,字体库就在该源码中,这是一种情况,有的网站会采用外部链接的方式加载字体库,原理一样. 并且从中可以看到是改字体库采用的是base64加密的方式.利用正则表达式将加密的内容提取出来,方便分析; 代码如下:

  1. 将加密的字体库提取出来候进行base64解密, 保存为xxx.woff格式字体库文件.用fontforge打开,查看其中自定义的字体.

可见,自定义的文字就这点,我们需要建立一个字典,将每个字的uni编码与文字对应起来,建立一个uni-> 文字的映射关系.然后利用python的fontTools字体处理库, 提取网站源码与源码的映射关系.具体在cmap标签下.

这样看起来就和原网址一样了,比如在 , 其中加密的 &#xe6aa, 在cmap中找到对应的uni码为 uni7537, 进一步在uni与文字的映射中找到文字为”男”, 这样就找到对应的性别为男,其他的加密也就可以破解了.

实际中遇到的问题

  1. 在用xpath提取到的信息中,提取加密的字符不是源代码中的字符,而变成了unicode格式的,这样就不能映射匹配了,

目前没有找到怎么提取出来的加密的信息和源代码一样,我就换了一种思路,在提取信息前,将源代码中的加密信息全部还原成文字.这只是一种替代的方法, 相比较还原关键信息,将整个源代码中的加密信息还原的工作量大,性能可能会受到影响.

关键性的映射关系代码

一下代码,只是针对某一特定网站的的解密,仅对代码思路进行参考,不适用于所有网站.其他网站请自行修改.


import base64
import re

from fontTools.ttLib import TTFont


def _get_base64_str(text):
    """
    获取响应中的base64字体字符串
    :param resp:
    :return:
    """
    pattern = re.compile(r'base64,(.*?)\) ')

    base64_str = pattern.search(text).group(1)
    # print(base64_str)

    return base64_str


def hex2word(resp):
    """
    将base64字符串加密的字体,转换成16进制-文字的匹配关系
    :param base64_str:
    :return: 替换后的结果
    """
    # 获取响应中字体部分
    base64_str = _get_base64_str(resp)
    # 手工解析字体uni与汉字对应的编码
    uni_word_dict = {
        "uni31": "1",
        "uni42": "B",
        "uni4EE5": "以",
        "uni30": "0",
        "uni6768": "杨",
        "uni9648": "陈",
        "uni535A": "博",
        "uni5E94": "应",
        "uni9AD8": "高",
        "uni5C4A": "届",
        "uni7537": "男",
        "uni41": "A",
        "uni5468": "周",
        "uni6280": "技",
        "uni65E0": "无",
        "uni5F20": "张",
        "uni4d": "M",
        "uni4E2D": "中",
        "uni37": "7",
        "uni5973": "女",
        "uni4E13": "专",
        "uni79D1": "科",
        "uni58E8": "士",
        "uni5434": "吴",
        "uni36": "6",
        "uni33": "3",
        "uni45": "E",
        "uni32": "2",
        "uni5927": "大",
        "uni7ECF": "经",
        "uni672C": "本",
        "uni7855": "硕",
        "uni5218": "刘",
        "uni38": "8",
        "uni39": "9",
        "uni674E": "李",
        "uni4E0B": "下",
        "uni6821": "校",
        "uni34": "4",
        "uni8D75": "赵",
        "uni738B": "王",
        "uni751F": "生",
        "uni35": "5",
        "uni9EC4": "黄",
        "uni9A8C": "验"
    }

    # 加密字体解密
    content = base64.b64decode(base64_str)
    # 保存为.woff文件
    with open("font_tmp.woff", "wb") as file:
        file.write(content)
    # fontTools.TTFont加载字体文件
    ttf = TTFont("font_tmp.woff")
    # 获取字体文件的cmap映射关系
    cmap = ttf.getBestCmap()
    # 处理映射关系,使映射的key与源码中使用的加密格式匹配
    cmap = {hex(key).replace("0x", "&#x"): value for key, value in cmap.items()}

    # 处理原码中加密部分进行还原
    for code, uni in cmap.items():
        if code in resp:
            resp = resp.replace(code+";", uni_word_dict[cmap[code]])

    # 返回还原后的源码
    return resp

总结

  1. 网站采用字体反爬虫,可以解析字体库文件,手工建立uni和文字映射关系;
  2. 利用fontTools库,分析字体库文件.woff/.ttf的cmap, 找到源码中加密与uni的映射
  3. 利用两个映射关系进行还原文字

问题:

  1. 如果网站中字体库文件不变,则只需要提取一次字体库文件即可; 即只需要上面的步骤1,2只操作一次即可.
  2. 如果网站的字体库文件中的cmap映射关系,每次请求都在变化,则需要每次请求,上面的1,2,3步骤都要重新做一遍
  3. 如果网站的字体库文件cmap和uni与文字的映射关系都在变化, 上面的方法不适用,原因在与手工建立uni与文字的对应关系这一步.理论上有这种情况,但是实际上如果是这样,那么网站开发者需要为一个字建立很多uni,这样后续的工作量会很大,所以一般网站不会有大量的字体库文件.前两种情况能处理绝大部分的字体反爬虫的网站了.
